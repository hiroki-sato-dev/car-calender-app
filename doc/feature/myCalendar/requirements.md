# マイカレンダー機能 要件定義

## 概要
ユーザーが自分専用のカレンダーを持てるようにする。
シェアカレンダーとは独立した個人用カレンダーで、LINEの個別チャットから予定登録ができる。

---

## 機能要件

### マイカレンダー

- 1ユーザーにつき1個作成可能
- カレンダー名は固定で「マイカレンダー」
- 作成後の削除はUI上に設けない（開発者がDB直接操作で削除するケースは想定する）
- 将来的に他ユーザーへの共有に拡張できる設計にする

### カレンダー一覧ページ

- 共有カレンダーセクションの上に「マイカレンダー」セクションを追加
- 未作成の場合：「作成する」ボタンを表示
- 作成済・LINE未連携の場合：
  - マイカレンダーへのリンクカード
  - LINE連携コード表示
  - コピーボタン（押すと「コピーしました」フィードバック）
  - 「LINE連携: XXXXXX とマイカレンダーBotに送ってください」の案内文
- 作成済・LINE連携済の場合：
  - マイカレンダーへのリンクカード
  - 「連携済み」バッジ表示

### ユーザーメニュー

- LINE連携済みの場合のみ「LINE連携を解除」を追加

### マイカレンダー詳細ページ（新規）

- iOS純正カレンダーアプリ風のUI
- 上部：FullCalendarによる月表示
- 下部：選択した日付の予定リスト（タイトル + 時間）
- 右上に「＋」ボタン → 選択日で予定作成モーダルを開く
- 予定タップ → 詳細表示モーダル → 「編集」ボタンで編集モーダルへ
- 共有カレンダーにある以下のボタンは非表示：
  - 登録コード表示
  - LINE通知設定
  - カレンダーを削除

## LINEマイカレンダーBot

### Botの分離

- 既存の共有カレンダーBotとは別のLINE Channelで運用
- 共有カレンダーBot：グループチャット専用（既存のまま）
- マイカレンダーBot：個別チャット専用（新規）

### LINE連携フロー

1. アプリのカレンダー一覧で連携コードを確認・コピー
2. マイカレンダーBotに「LINE連携: XXXXXX」と送信
3. BotがlineUserIdとアプリのUserを紐付けて保存
4. 「連携しました！」と返信

### リッチメニュー

- 全ユーザーに表示（未連携ユーザーにも表示）
- ボタン構成（3ボタン）：
  - 予定登録：会話フロー開始
  - 使い方：使い方テキストを返信
  - カレンダー：アプリのマイカレンダーページへ遷移（未ログインはリダイレクト）
- デザイン：グレー系モダンデザイン、SVGからPNG自動生成

### 予定登録の会話フロー

```
ユーザー: 「予定登録」
Bot: 「タイトルを教えてください」
ユーザー: 「買い物」
Bot: 「日時を教えてください　例: 明日10時〜11時、3/20 9:00-10:00」
ユーザー: 「明日10時〜11時」
Bot: 「3月20日 10時〜11時で「買い物」を登録しました！」
```

- タイトル → 日時 → 登録完了の3ステップ
- 会話の状態はDBで管理
- 日時パース：正規表現ベースで主要な日本語表現に対応
- パース失敗時は例文付きで再入力を促す

### 個別チャットの処理分岐

```
未連携ユーザー
  → 連携コード（LINE連携: XXXXXX）にマッチ → 連携処理
  → それ以外 → 連携を促す案内メッセージ

連携済みユーザー
  → 「予定登録」 → 会話フロー開始
  → 「使い方」 → 使い方テキスト返信
  → 会話フロー中 → 対応するステップの処理
  → それ以外 → 案内メッセージ
```
